user functions, channel functions and message functions. These correspond
to the various user stories we have developed and are also summarized below. 

We need to authenticate the user first before building the channels. 
Start with the auth functions, then lead into the user functions in order to complete each usersâ€™ profiles.

Then we will go on to build channels where users can join and leave, view the members of each channel and invite people.

Then the message functions come next, the users may leave messages, view messages, edit and react to them. 

The search function will then be implemented, allowing users to search for a string 
with which slackr will attempt to match with an existing message.

Finally, we will develop the standup functions, 15-minute period where users 
send messages and then have them summarized by the end.
